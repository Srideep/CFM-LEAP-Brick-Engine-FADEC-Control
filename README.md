# FADEC Project

## Full Authority Digital Engine Control for a LEGO-Built CFM LEAP Turbofan

A hobby-scale implementation of an aircraft engine FADEC (Full Authority Digital Engine Control) system using a LEGO Mindstorms EV3, a brick-built CFM LEAP turbofan engine model, and MATLAB/Simulink.

The FADEC reads a physical throttle lever, controls fan spool speed via a closed-loop PID controller, simulates engine thermodynamics in real time, and enforces acceleration, EGT, and overspeed protection limits — exactly like the real system on a CFM LEAP-1B engine.

## What Is FADEC

FADEC (Full Authority Digital Engine Control) is the system that provides complete digital management of an aircraft engine's performance. It takes pilot inputs (throttle lever position) plus sensor data (temperatures, pressures, shaft speeds) and computes the optimal fuel flow and actuator positions in real time.

"Full authority" means the digital computer has sole control — there is no mechanical backup. The system continuously monitors engine parameters and adjusts fuel metering, bleed valves, and variable geometry to keep the engine within safe limits while maximizing performance.

Key functions of a FADEC include preventing the pilot from exceeding engine limits (over-temperature, over-speed, compressor stall), optimizing fuel efficiency, simplifying engine starting, and enabling engine-to-engine thrust matching on multi-engine aircraft.

This project implements these core FADEC functions at hobby scale using a LEGO motor as the "engine" and MATLAB/Simulink as the control computer.

---

## Project Overview

The system consists of three layers.

**Physical layer:** A brick-built CFM LEAP turbofan engine model (MOC-133571 by Levihathan, 1659 pieces) with an EV3 Large Motor driving the fan shaft through a gear train. A second motor actuates the thrust reverser cowl. A third motor serves as a throttle lever input, reading its built-in encoder to determine lever position.

**Control layer:** A LEGO Mindstorms EV3 Intelligent Brick (31313) running compiled C code generated by Simulink. The EV3 reads motor encoders at 50 Hz, executes the PID control law and limit logic, and drives the fan motor. It operates standalone after deployment — no PC required.

**Development layer:** MATLAB/Simulink on a PC for model development, PID tuning, dashboard visualization, and code deployment to the EV3. In external mode, the PC and EV3 run simultaneously for live tuning with graphical scopes and sliders.

### What's in the EV3 31313 Set

The retail Mindstorms EV3 set includes the EV3 Intelligent Brick, 2× Large Motors (with 1° encoders), 1× Medium Motor (with encoder), 1× Touch Sensor, 1× Color Sensor, 1× IR Sensor + IR Remote, 6× AA battery holder, and approximately 550 Technic pieces.

### Motor Assignments

| EV3 Port | Motor Type | FADEC Role | Mode |
|----------|------------|------------|------|
| Port A | Large Motor | N1 fan spool drive | Speed-controlled (PID) |
| Port B | Medium Motor | Thrust reverser cowl | Position-controlled |
| Port C | Large Motor | Throttle lever input | Encoder read only (not driven) |
| Port 1 | IR Sensor | Engine start / remote | Input |

### Mechanical Integration

The CFM LEAP kit ships with a Power Functions M-motor. This must be replaced with the EV3 Large Motor from the 31313 set. The EV3 motor is physically larger, so 5–10 bricks around the motor bay may need modification. The Technic cross-axle output shaft is compatible across both systems.

Motor C is mounted externally in a small Technic bracket frame. A long Technic lever arm attaches to its axle, functioning as a throttle quadrant. The lever travel maps to throttle demand: full aft = IDLE, midway = CLB, full forward = TOGA.

## How the PID Controller Works

The PID controller is the core of the FADEC. It determines how much power to send to the fan motor to achieve the demanded N1 speed.

**P (Proportional)** looks at the current gap between demanded N1 and actual N1. A large gap produces a large motor command. As the engine approaches the target, the command naturally decreases.

**I (Integral)** accumulates the error over time. Even if the current gap is small, the integral term builds up and adds additional command until the gap is completely eliminated. This ensures zero steady-state error.

**D (Derivative)** monitors how quickly the gap is changing. If N1 is approaching the target rapidly, the derivative term applies braking to prevent overshoot.

### Tuned Gains

| Parameter | Value | Role |
|-----------|-------|------|
| Kp | 3.5 | Fast response to throttle changes |
| Ki | 1.5 | Eliminates steady-state error within ~2 seconds |
| Kd | 0.15 | Light braking to prevent overshoot |

Found using `fadec_pid_tuner.m`. Optimized for fast spool-up with minimal overshoot — matching real FADEC priorities where exceeding limits is worse than slow response.

## Thermodynamic Model

Since the brick-built engine has no real combustion, the FADEC simulates engine thermodynamics using polynomial curve fits based on public CFM LEAP-1B data.

| Parameter | Formula | At Idle (~19%) | At Max (~100%) |
|-----------|---------|----------------|----------------|
| EGT (°C) | 0.12·N1² + 1.5·N1 + 350 | ~410 | ~1,250 |
| Ps3 (psia) | 0.004·N1² + 0.1·N1 + 14.7 | ~16 | ~55 |
| Wf (kg/hr) | 45·N1 − 500 | ~355 | ~4,000 |
| Fn (kN) | 0.013·N1² | ~4.7 | ~130 |

## Limit Protection Logic

Three protection layers, evaluated every cycle:

**1. Acceleration/Deceleration Limiter** — Clamps dN1/dt to +8%/sec (accel) and -12%/sec (decel). Prevents compressor surge during rapid throttle advances.

**2. EGT Redline Protection** — When EGT exceeds 1030°C (30° margin below 1060°C redline), N1 is proportionally reduced at 0.5% per °C above the margin. Never reduces below idle.

**3. Overspeed Protection** — Hard cap at N1 = 104%. Last line of defense.

Each layer produces a status flag displayed on the dashboard as amber/red indicator lamps.

## Engine Start Sequence

| State | What Happens | Transition |
|-------|-------------|------------|
| OFF | Waiting for start button | Start pressed → CRANKING |
| CRANKING | Motor ramps at 6% PWM/sec, max 30%. No fuel. | N1 ≥ 12% → LIGHTOFF |
| LIGHTOFF | Fuel valve opens. EGT spikes. Start EGT limit 725°C monitored. | N1 ≥ 15% → STABILIZE |
| STABILIZE | PID takes over. Engine settles to idle. | Stable ±1% for 3s → IDLE |
| IDLE | Normal FADEC operation. Full throttle authority. | N1 < 2% → OFF |

Crank timeout: 10 seconds. Hot start (EGT > 725°C): automatic abort.

## Setup Instructions

### 1. Software Setup (PC — no hardware needed)

```matlab
% 1. Install MATLAB R2021b+ with Simulink
% 2. Install EV3 support package (Add-Ons → Get Add-Ons → search "EV3")
% 3. Copy all .m files to a single folder, cd to it
% 4. Run:
fadec_setup          % Load parameters
fadec_sim_test       % Verify simulation (produces 8-panel plot)
fadec_pid_tuner      % Explore PID gains
```

### 2. Hardware Assembly

1. Build CFM LEAP engine model (MOC-133571)
2. Replace PF M-motor with EV3 Large Motor (modify ~5-10 bricks for fit)
3. Mount EV3 Medium Motor near thrust reverser cowl with worm gear linkage
4. Build Technic bracket for Motor C throttle lever
5. Connect motors to EV3 brick: Port A (fan), Port B (reverser), Port C (throttle)
6. Install 6× AA batteries

### 3. Deploy

```matlab
fadec_setup               % Load parameters
fadec_build_model         % Follow the printed block wiring guide
% Wire Simulink blocks, set mode to External, click Run
% Tune PID live, then deploy standalone
```

## Calibration

After assembly, calibrate four things:

**Throttle lever range:** Move lever full aft → full forward, note encoder degrees. Update `tla_schedule.lever_deg` in `fadec_setup.m`.

**N1 speed scaling:** Determine what motor speed maps to 100% N1. Adjust the thermo model if needed.

**PID gains:** Tune live in Simulink external mode. Start with simulation-tuned values (3.5, 1.5, 0.15) and refine.

**Reverser travel:** Find the motor B positions that fully open/close the cowl. Update `reverser.deploy_position` and `reverser.stow_position`.

## Simulation Results

The 60-second simulation exercises idle → climb → TOGA → cruise → slam → snap idle → re-advance.

With tuned parameters (Kp=3.5, Ki=1.5, Kd=0.15, egt_a2=0.12, B_viscous=0.0003):

- N1 tracks demand with ~2-second rise time and minimal overshoot
- Acceleration limiter fires on all rapid throttle transitions
- EGT protection activates during sustained high-power, holding EGT below 1060°C redline
- Peak thrust ~85 kN (65% of rated LEAP-1B output)
- No overspeed events during normal operation

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Plots appear all black | MATLAB dark theme | `set(0, 'DefaultFigureColor', 'w')` before running |
| N1 never reaches demand | PID too weak or friction too high | Increase Kp or reduce B_viscous in ev3_motor_plant.m |
| N1 oscillates around target | Kp too high or Kd too low | Reduce Kp or increase Kd |
| EGT protection never triggers | Thermo coefficients conservative | Increase thermo.egt_a2 in fadec_setup.m |
| EV3 won't connect | WiFi/USB issue | Check firmware version matches support package |
| Motor doesn't spin | Wrong port | Verify cable on Port A matches Simulink block |
| Throttle lever feels stiff | Motor C being driven | Ensure Simulink only reads encoder, no PWM write |
